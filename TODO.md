# TODO / архитектурные заметки

Список отложенных решений и точек для будущего рефакторинга. Не публикуется на
сайт (лежит в корне проекта, вне `docs_dir: wiki/`), хранится в git как памятка.

## Навигация: явный `nav:` с дублированием дерева

**Текущее решение (осознанное):** навигация сайта задаётся явным списком `nav:`
в `mkdocs.yml`. Он повторяет структуру каталога `wiki/` — каждая страница
перечислена вручную. Да, это дублирование дерева.

**Почему так сейчас:**

- **Порядок — смысловой, а не алфавитный.** Разделы идут «Процесс → Предусловия →
  Проблемы → Особые случаи → Справочное», внутри — по логике процесса
  (Портал → Кабинет → Этапы → Оплата…). Автогенерация упорядочила бы по имени
  файла (алфавит) и всё бы поехало.
- **Русские названия разделов**, отличные от имён файлов/папок
  (напр. «Когда что-то идёт не так» для папки `problems/`).
- **`mkdocs build --strict` ловит забытые страницы** — если файл не упомянут в
  `nav:`, сборка предупреждает. Полезная страховка при ручном наполнении.

Цена — при добавлении статьи нужно дописать строчку в `nav:`. На ~20 страницах
это терпимо.

## Два способа уйти от дублирования при разрастании вики

Когда страниц станет заметно больше и ручной `nav:` начнёт мешать — есть два пути.

### Вариант 1. Убрать `nav:` совсем (полная автогенерация)

Если удалить блок `nav:` из `mkdocs.yml`, MkDocs строит навигацию из дерева
`wiki/` автоматически.

- ➕ Нулевая поддержка, никакого дублирования.
- ➖ Порядок — **алфавитный по имени файла** (смысловой порядок теряется; можно
  частично лечить числовыми префиксами имён файлов, но это шумит в URL).
- ➖ Названия разделов берутся из имён папок / `index.md`, меньше контроля.
- ➖ `--strict` больше не подсветит «потерянную» страницу — она просто молча
  появится где-то в меню.

### Вариант 2. Плагин `mkdocs-awesome-nav` (золотая середина)

Наследник awesome-pages: <https://github.com/lukasgeiter/mkdocs-awesome-nav>.
Дерево остаётся источником истины, а порядок/названия задаются **маленькими
файлами `.nav.yml` рядом с контентом** — центральной простыни в `mkdocs.yml`
больше нет. Можно перечислять руками только то, что нужно закрепить, а остальное
подхватывать через `- ...`.

- ➕ Полный контроль порядка и названий без единого длинного списка.
- ➕ Порядок конфигурируется локально, по папкам.
- ➖ Нужен **PyPI-пакет (pip)** — в apt его нет, а `mkdocs-material` в этом
  окружении ставился через apt. Установка pip-зависимости — по правилам проекта
  только с явного разрешения пользователя.
- ➖ Ещё одна зависимость в цепочке сборки/деплоя.

**Рекомендация:** пока структура стабильна и страниц немного — оставить явный
`nav:`. При активном росте вики предпочесть Вариант 2 (`awesome-nav`).
